# cppcheck suppressions for the MISRA-C:2012 analysis (make analyze-misra).

# This file holds DOCUMENTED, PROJECT-SPECIFIC deviations: MISRA rules we
# knowingly do not satisfy in our own source. Every entry maps to a numbered
# deviation record ("D-n") in MISRA_COMPLIANCE.md, which carries the rationale.
# A suppression here without a matching D-record in that document is a defect --
# the whole point of MISRA compliance is that every deviation is justified.

# Scope: suppressions are PER FILE (errorId:filename), not project-wide. A new
# instance of one of these rules in a NEW file therefore still surfaces and
# fails `make analyze-misra`, forcing a conscious review + a new deviation
# rather than silently inheriting a blanket waiver.

# Out of scope (handled on the cppcheck command line in the Makefile, not here):
# violations inside the avr-libc / avr-gcc system headers -- the toolchain
# library is outside this project's compliance boundary.

# (cppcheck parses a line that is exactly "#" as a malformed suppression, so
# blank separator lines below are left empty rather than commented.)

# --- D-1: hardware register access (Rules 11.4, 10.1, 10.8) ------------------
# Reading/writing AVR I/O registers through avr-libc's _SFR_* macros inherently
# converts an integer address to a volatile pointer (11.4) and performs bit
# operations / composite casts in register read-modify-write (10.1, 10.8).
# Unavoidable for bare-metal AVR. See MISRA_COMPLIANCE.md D-1.
misra-c2012-11.4:src/bypass_mcu_avr_classic.c
misra-c2012-11.4:src/bypass_output_cd4053_simple.c
misra-c2012-11.4:src/bypass_output_cd4053_with_mute.c
misra-c2012-11.4:src/bypass_output_tq2_l2_5v_relay.c
misra-c2012-10.1:src/bypass_mcu_avr_classic.c
misra-c2012-10.1:src/bypass_output_cd4053_simple.c
misra-c2012-10.1:src/bypass_output_cd4053_with_mute.c
misra-c2012-10.1:src/bypass_output_tq2_l2_5v_relay.c
misra-c2012-10.8:src/bypass_mcu_avr_classic.c

# --- D-2: cross-translation-unit shared pin macros (Rule 2.5) ----------------
# The per-MCU pin map (bypass_pins_avr_classic.h / bypass_pins_pic10f32x.h) is
# the single source of truth for every variant's pin assignment: FOOTSW_PIN /
# LED_PIN (used by the core), each variant's control pins (CD4053_PIN;
# RELAY_SET/RESET_PIN; CD4053_CTL1/2 -- each used only by its own driver), and
# BYPASS_OUTPUT_DDR_MASK. cppcheck analyzes each TU independently, so any pin a
# given TU does not reference is reported as "unused". They are not dead code --
# each is used by some build. See D-2.
# PRESSED_THRESH / RELEASE_THRESH are defined in bypass_config.h and used by
# bypass_pure.c and bypass_mcu_avr_classic.c, but not by the output driver TUs that
# nonetheless include bypass_config.h for their static_assert guards.
# The PIC pin map (bypass_pins_pic10f32x.h) is flagged for the SAME reason in the
# PIC MISRA run (make pic-analyze-misra); additionally, the unselected AVR pin map
# is reachable via bypass_output_common.h's #elif, so cppcheck records ITS macros
# in the cross-config directive list and reports them "unused" too -- already
# covered by the bypass_pins_avr_classic.h entry above.
misra-c2012-2.5:src/bypass_pins_avr_classic.h
misra-c2012-2.5:src/bypass_pins_pic10f32x.h
misra-c2012-2.5:src/bypass_config.h
